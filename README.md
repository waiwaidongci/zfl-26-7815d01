# 夜间灯塔守望

直接打开 `index.html` 即可游玩。方向键旋转光束，空格切换光束宽度，护送所有船只抵达右侧港湾。

## 游戏玩法

- **← / →** 旋转光束
- **空格** 切换光束宽度（宽/窄）
- **P** 暂停游戏
- **R** 重开本关

## 关卡系统

游戏内置 3 个关卡，难度递进。通关后自动解锁下一关。

### 自定义关卡

在主菜单点击「关卡编辑器」可创建和编辑自定义关卡：
- 放置暗礁和雾区
- 调整船只数量和起始位置
- 设置港湾位置
- 配置天气事件（海风、浓雾、灯塔闪烁）
  - 海风：船只漂移加剧，可调整持续时间、冷却时间、漂移强度和速度加成
  - 浓雾：光束射程缩短，可调整持续时间、冷却时间、射程缩减和雾气浓度
  - 灯塔闪烁：光束角度抖动，可调整持续时间、冷却时间、抖动幅度和闪烁强度
  - 开启天气后点击「试玩关卡」可立即体验当前配置的效果
- 保存后的天气配置在以下场景中正确生效：
  - **选择关卡**：关卡卡片显示天气图标，进入关卡时加载天气
  - **航线战役**：战役进度会根据难度倍率调整天气持续时间与冷却时间
  - **自定义航线**：章节卡片聚合显示所含关卡的实际天气类型，而非根据难度硬编码
  - **关卡包导入导出**：天气配置随关卡数据一起导出和导入，保留完整参数
  - **回放记录**：回放中录制 `weatherStart`/`weatherEnd` 事件，播放时实时渲染雾气覆盖、风线、光束射程缩减和闪烁效果

### 自定义航线

在「航线战役」界面点击「🧭 自定义航线」可编排专属连续挑战航线，将自定义关卡串联成独特冒险。

#### 编排航线

1. 点击自定义航线卡片进入编辑器
2. 在「选择关卡」列表中点击关卡添加到航线
3. 已选关卡会显示在下方，可进行以下操作：
   - **调整顺序**：点击 ↑/↓ 按钮上下移动关卡位置
   - **移除单个**：点击 ✕ 按钮移除单个关卡
   - **一键清理**：若有关卡已失效，可点击「🧹 一键清理失效关卡」批量移除

#### 航线信息

编辑器实时显示航线统计信息：
- **关卡数**：已选关卡总数（失效关卡会单独标注）
- **初始生命**：根据关卡数量自动计算（每 2 关 +1 生命，最少 1 颗心，最多 5 颗心）
- **船只总数**：所有关卡的船只数量之和

#### 失效关卡处理

如果某个自定义关卡已被删除，航线编辑器会自动识别并提示：
- 失效关卡以红色边框显示，标注「⚠️ 已失效」
- 章节选择页的航线卡片会显示 ⚠️ 警告图标和提示文字
- 战役开始前会提示失效关卡数量，开始后将自动跳过
- 编辑器提供「一键清理」按钮，可快速移除所有失效关卡

#### 保存与开始

- 点击「确认并开始」保存航线并立即开始战役
- 航线配置自动保存至本地，下次打开可继续使用或重新编排
- 战役进度会单独保存，中途退出可随时继续

### 关卡包导入导出

支持将自定义关卡打包导出为文件，或导入他人分享的关卡包。

#### 导出关卡包

1. 进入「选择关卡」界面
2. 点击「📤 导出关卡包」按钮
3. 填写关卡包名称、作者、说明
4. 勾选要导出的关卡（支持全选/全不选）
5. 点击「导出文件」，浏览器会自动下载 `.zfl26pack.json` 文件

也可以在编辑器中点击「📤 导出当前关卡」快速导出单个关卡。

#### 导入关卡包

1. 进入「选择关卡」界面
2. 点击「📥 导入关卡包」按钮
3. 选择导入方式：
   - **从文本粘贴**：粘贴关卡包 JSON 数据后点击「解析数据」
   - **选择文件**：选择本地 `.zfl26pack.json` 文件
4. 解析成功后可查看关卡包信息与关卡预览列表，每个关卡显示：
   - 🚢 船只数量、🪨 暗礁数量、🌫️ 雾区数量
   - 天气摘要（🌬️海风 / 🌫️浓雾 / ⚡闪烁 / 🌤️晴朗）
   - 最佳分
   - ⚠️ 重复状态标记（与本地关卡同 ID 或同名时显示）
5. 勾选要导入的关卡（支持「全选」/「全不选」，默认全选）
6. 选择重复关卡处理策略：
   - **跳过重复**：保留本地已有的同 ID 或同名关卡（默认）
   - **覆盖重复**：用导入的关卡数据和最佳分覆盖本地
   - **重命名导入**：为重复关卡生成新 ID，保留两者
7. 点击「确认导入」完成
8. 导入完成后，关卡选择、自定义航线可选关卡及最佳分显示会立即刷新

## 玩家档案导入导出

支持将完整玩家档案打包导出为文件，或从他人分享的档案文件中选择性导入数据。导入过程完全原子化——失败时自动回滚到导入前状态，不会破坏本地已有存档。

### 导出玩家档案

1. 进入「游戏设置」界面
2. 点击「📤 导出档案」按钮
3. （可选）填写玩家名称，便于识别档案归属
4. 勾选要导出的数据类别（默认全选），共 10 类：
   - 🗺️ 自定义关卡
   - 📦 关卡包来源信息
   - 🏆 单关最佳分
   - ⚔️ 战役进度
   - 🎖️ 战役奖励
   - 🏅 成就
   - 📊 统计
   - 📜 航海日志
   - 🎬 回放摘要
   - 🧭 最近自定义航线
5. 点击「导出文件」，浏览器自动下载 `.zfl26profile.json` 文件

### 导入玩家档案

1. 进入「游戏设置」界面
2. 点击「📥 导入档案」按钮
3. 选择导入方式：
   - **从文本粘贴**：粘贴档案 JSON 数据后点击「解析数据」
   - **选择文件**：选择本地 `.zfl26profile.json` 文件
4. 解析成功后查看档案信息：玩家名、导出时间、游戏版本、格式版本、包含类别数
5. **变更预览**：界面会显示每类数据的处理方式：
   - ➕ **新增**：本地不存在的数据项
   - 🔄 **覆盖**：本地已有但导入版本更优的数据（如更高的最佳分）
   - ⏭ **跳过**：本地数据不逊于导入版本的数据
   - 🔀 **合并**：统计数据按最大值与累加智能合并
6. 勾选要导入的数据类别（可只导入其中几类）
7. 点击「确认导入」完成
8. 导入成功后，关卡选择、战役面板、成就统计、航海日志、战绩分析等所有面板会立即刷新并读取到一致的新数据

### 导入原子性保证

- 导入开始前，系统会对全部本地存档数据创建完整快照
- 导入过程中任何类别出现错误，立即触发回滚，所有数据恢复到快照状态
- 只有全部类别的数据写入成功后，导入才算完成
- 失败时会显示具体错误信息，本地数据不受任何影响

### 智能合并策略

不同数据类别采用不同的合并策略，确保导入不损失已有成果：

| 数据类别 | 合并策略 |
|---------|---------|
| 自定义关卡 | 同 ID/同名覆盖，否则新增 |
| 单关最佳分 | 取两者较高值 |
| 战役章节最佳分 | 取两者较高值 |
| 战役通关标记 | 逻辑或（任意一方已通关即保留通关） |
| 战役奖励称号 | 去重合并 |
| 成就解锁 | 逻辑或合并 |
| 统计（累计护送等） | 数值累加 |
| 统计（最大连败、最快通关） | 取最优值 |
| 天气胜场/失败原因 | 数值累加 |
| 航海日志 | 按 ID 去重合并 |
| 回放数据 | 按 ID 去重合并 |
| 自定义航线 | 导入覆盖本地 |

### 数据迁移说明

- **浏览器/设备迁移**：在旧设备导出档案 → 在新设备导入档案，可完整迁移所有游戏进度
- **格式版本兼容**：低版本档案可在高版本游戏中正常导入；版本过新时会提示升级游戏
- **部分迁移**：导入时可只勾选需要的类别，例如只迁移自定义关卡而保留本地成就
- **存档备份**：定期导出档案作为备份，可防止浏览器清理 localStorage 导致进度丢失
- **与关卡包的关系**：玩家档案包含自定义关卡数据，也包含每一关来自哪个关卡包的来源信息；导入关卡包时会自动记录来源，导出档案时一并带出

## 成就与统计系统

成就系统会自动追踪护航、战役和天气表现，所有数据保存在本地 `localStorage` 中。旧版本统计会在读取时自动补齐新增字段，既保留已有成就进度，也支持新增的战役、天气和失败原因数据。点击「重置成就与统计」会清空成就解锁状态，并把统计重置为包含新增字段的完整默认结构。

### 游戏统计

- **累计护送**：成功护航的船只总数
- **完美通关**：无船只损失的通关次数
- **窄光束通关**：全程未切换宽光束的通关次数
- **最快通关**：历史最快通关时间
- **当前/最大连败**：连续失败次数记录
- **战役通关**：完成战役章节的次数
- **无损战役**：未损失战役生命完成章节的次数

### 天气与失败分析

- **天气胜场**：分别记录海风、浓雾和灯塔闪烁天气出现过的胜利次数
- **最常见失败原因**：统计偏离上侧、偏离下侧、光束迷航、撞上暗礁四类失败原因

### 新增成就

- **初战告捷**：完成第一个战役章节
- **战役老兵**：累计完成 3 次战役章节
- **无损凯旋**：不损失任何一条生命完成战役章节
- **雾中领航**：在浓雾天气影响下成功通关
- **闪耀护送**：在灯塔闪烁期间成功护送船只抵达港湾
- **自由航路**：完成自定义航线战役

## 每日挑战模式

在主菜单点击「📅 每日挑战」进入每日挑战模式。每天根据日期种子生成一条固定航线，关卡由内置关卡和你的自定义关卡池组合生成，并附带当天的天气组合、生命限制和计分倍率。

### 核心机制

- **日期种子**：使用 `YYYYMMDD` 格式作为伪随机种子，同一天无论刷新页面还是重新进入，生成的挑战完全相同
- **关卡选择**：从所有已有关卡（内置 + 自定义）中随机选取 3~5 个，组成当日航线
- **天气组合**：每天随机激活 1~3 种天气事件（海风、浓雾、灯塔闪烁），作用于航线所有关卡
- **生命限制**：每天 2~4 条生命，船只失事扣 1 生命，生命归零则挑战失败
- **计分倍率**：每天 1.2x~2.0x 的得分倍率，乘以基础抵达分
- **航线名称**：每天随机一个诗意航线名（如"晨曦航线"、"暮色航道"等）

### 每日挑战流程

1. 进入「每日挑战」面板查看今日航线信息（航线名、关卡数、天气、生命、倍率）
2. 点击「开始今日挑战」进入连续关卡模式
3. 船只失事扣生命，可重试本关；生命归零则挑战失败
4. 通关所有关卡后显示结算面板（最终得分、剩余生命、今日最佳）
5. 刷新纪录时显示「🎉 新纪录！」提示

### 进行中的挑战

- 中途退出游戏，进度自动保存至 `localStorage`
- 重新进入「每日挑战」面板，会显示进行中的挑战（进度、生命、得分）
- 支持「继续」「重开」「放弃」三种操作

### 历史日历记录

- 面板下方显示月历，标记每天挑战状态：
  - 🟢 绿色底：已通关
  - 🟡 黄色底：已尝试但未通关
  - 🔲 金色边框：今日
  - 灰色：未尝试 / 未来日期
- 每个日期显示当日最佳分数
- 可切换月份查看历史记录
- 统计区显示：连续通关天数、累计通关数、历史最高分

### 航海日志标记

每日挑战的航海日志条目会自动标记为「📅 每日」标签，并记录对应的日期。在战绩分析面板中可通过来源筛选查看。

### 回放保存

每日挑战的每一关都会自动保存回放数据，可在航海日志详情中查看回放。

### 日期边界规则

- **时区**：日期使用浏览器本地时区（`new Date()` 的 `getFullYear()`/`getMonth()`/`getDate()`）
- **日期切换时间**：本地时间 00:00（午夜）即进入新一天的挑战
- **确定性保证**：
  - 同一天、同一关卡池 → 生成完全相同的挑战（航线、天气、生命、倍率）
  - 自定义关卡的增删会影响关卡池，从而导致生成不同的关卡序列
  - 内置关卡不会变化，因此即使自定义关卡池为空，每天也有固定挑战
- **跨日保存**：进行中的挑战绑定日期，跨日后旧挑战自动失效，需从新一天的挑战开始
- **重置数据**：「重置每日挑战」会清除所有历史记录和进行中的挑战，不影响其他模式

## 其他功能

- **战绩分析**：深入分析每一次航行的表现
  - 支持按关卡名称搜索
  - 支持按失败原因筛选（偏离航道、光束迷航、撞上暗礁等）
  - 支持按得分和耗时排序
  - 详情展示天气事件时间线、船只损失原因
  - 标识是否来自战役模式，是否观看过回放
  - 提供回放入口和重新挑战功能
  - 统计摘要包含胜率、平均得分、平均用时、失败原因分布
- **回放系统**：支持查看对局回放，回放数据持久化存储

## 游戏设置

在主菜单点击「游戏设置」或游戏中点击右上角 ⚙ 按钮可打开设置面板，所有设置自动保存到本地。

### 基础设置

- **音效**：开关游戏音效
- **画面亮度**：30%-100% 可调，调节游戏画面整体亮度
- **光束旋转灵敏度**：0.3x-3.0x 可调，控制方向键旋转光束的速度
- **暂停键提示**：开关游戏底部的「按 P 暂停游戏」提示

### 可访问性选项

- **旋转方向反转**：反转左右方向键的旋转方向，适配不同操作习惯
- **降低闪烁效果**：大幅降低「灯塔闪烁」天气效果的光束抖动和明暗变化，适合对光敏感的玩家
- **高对比度海面**：使用更高对比度的海面配色，增强船只、暗礁等游戏元素的可视性
- **暂停时静音**：暂停游戏时自动静音，恢复游戏时自动恢复音效
- **回放自动暂停**：进入回放模式时自动暂停，便于从起始位置开始观看

设置在以下所有游戏模式中即时生效：普通游玩、战役模式、试玩关卡、回放展示。主设置面板和游戏内设置弹窗保持同步。

## 开发与验证流程

### 快速开始

**无需安装任何依赖，直接打开 `index.html` 即可游玩。

### 工程化验证流程

项目提供了轻量的工程化验证流程，用于开发过程中的快速回归验证。

#### 首次设置

```bash
# 安装依赖
npm install

# 安装 Playwright 浏览器（仅需首次）
npm run install:browsers
```

#### 日常开发

```bash
# 启动本地开发服务器（可选，用于浏览器访问 http://localhost:8080）
npm run dev
```

#### 回归验证

一条命令运行所有测试：

```bash
# 运行全部测试（单元测试 + E2E 测试）
npm run test:all
```

#### 单独运行测试：

```bash
# 仅运行单元测试（jsdom，无浏览器）
npm test

# 仅运行 E2E 测试（Playwright，无头模式）
npm run test:e2e

# 运行 E2E 测试（有头模式，可观察浏览器行为）
npm run test:e2e:headed
```

### E2E 测试覆盖

E2E 测试位于 `tests-e2e/` 目录，覆盖以下核心场景：

1. **页面加载验证**
   - Canvas 渲染正常
   - 主菜单所有关键按钮可见

2. **主菜单入口验证**
   - 关卡选择可进入并返回
   - 航线战役可进入并返回
   - 每日挑战可进入并返回

3. **关卡编辑器验证**
   - 可进入关卡编辑器
   - 点击「试玩关卡」按钮可启动游戏
   - 试玩后可返回编辑器

4. **设置持久化验证**
   - 切换音效设置后写入 localStorage
   - 刷新页面后设置保持不变

5. **导入错误处理验证**
   - 导入弹窗可打开
   - 粘贴无效 JSON 能正确解析错误
   - 错误信息正确显示

### 测试配置

- Playwright 配置文件：`playwright.config.js`
- 测试会自动启动本地 HTTP 服务器，无需手动启动
